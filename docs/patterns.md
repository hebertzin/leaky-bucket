## Main Design Patterns Used

To enhance flexibility and maintainability, the following design patterns are used:

- **Middleware**: Adds cross-cutting concerns like logging, authentication, and validation.
- **Factory**: Manages the creation of complex objects and dependencies.
- **Dependency Injection & Inversion of Control (DI/IoC)**: Promotes loose coupling and testability.
- **Singleton**: Ensures single instances of certain components (e.g., database connections).
- **Repository**: Abstracts the persistence logic and allows easy swapping of data sources.
